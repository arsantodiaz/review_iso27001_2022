Definisi serangan homograf

Serangan homograf (atau IDN Homograph Attack) adalah teknik penipuan di mana penyerang membuat domain/URL yang terlihat identik dengan situs asli, tetapi menggunakan karakter dari alfabet lain yang bentuknya sangat mirip.

Misalnya: huruf latin a vs karakter Sirilik а — tampak sama di layar, tapi berbeda secara teknis. Browser menerima keduanya sebagai domain valid!

Bagaimana cara kerjanya?
1
Penyerang memilih target
Misal: situs bank terkenal atau email korporat
2
Ganti karakter kunci
Satu atau dua huruf diganti karakter Unicode mirip dari bahasa lain
3
Daftarkan domain palsu
Domain baru didaftarkan dan terlihat sah di browser
4
Sebar via phishing
Link dikirim via email, chat, atau SMS untuk mencuri data login
Mengapa berbahaya?
!
Lolos pemeriksaan visual — mata manusia tidak bisa membedakan tanpa tools khusus
!
SSL certificate tetap valid — gembok hijau muncul, seolah-olah situs aman dan terpercaya
!
Bypass email filter — sistem keamanan sering tidak mendeteksi karena URL tampak normal
!
Efektif di mobile — layar kecil membuat URL semakin sulit dibaca dengan teliti
!
Menarget kredensial, data keuangan dan informasi sensitif perusahaan
Perbandingan URL asli vs palsu — coba bedakan!
ASLIhttps://www.google.comAman
PALSUhttps://www.gооgle.comBerbahaya

Pada contoh di atas, huruf "o" kedua dan ketiga di URL palsu menggunakan karakter Sirilik (U+043E), bukan huruf Latin biasa. Keduanya tampak identik!

ASLIhttps://paypal.com/loginAman
PALSUhttps://pаypal.com/loginBerbahaya

Huruf "a" pertama di URL palsu adalah karakter Sirilik. Identik secara visual, tapi domain berbeda sepenuhnya.

ASLIhttps://microsoft.comAman
PALSUhttps://mıcrosoft.comBerbahaya

Huruf "i" menggunakan karakter Turkish dotless-i (ı). Hampir tak terlihat bedanya di ukuran font kecil.

oLatin o (U+006F)

Cara mendeteksi URL homograf
Hover dulu sebelum klik — lihat URL yang muncul di status bar browser di pojok kiri bawah
Salin URL ke text editor — tempel di Notepad atau VS Code, karakter aneh kadang terlihat berbeda
Gunakan URL decoder tool — cek di whoisxmlapi.com atau tools serupa untuk decode karakter Unicode
Perhatikan "xn--" di address bar — browser modern mengonversi IDN homograf ke format Punycode seperti xn--googie-rya.com
Ketik manual langsung — jangan klik link dari email atau chat, ketik URL langsung di browser
Gunakan bookmark — simpan situs penting di bookmark agar tidak perlu mengetik atau klik link
Tanda-tanda URL mencurigakan
!
URL diterima via email tidak diminta, chat, atau SMS yang mendesak
!
Ada urgensi berlebihan — "akun Anda akan diblokir dalam 24 jam!"
!
Nama domain sedikit berbeda dari yang biasa — paypa1 bukan paypal, g00gle bukan google
!
TLD mencurigakan — .xyz, .tk, .ml alih-alih .com atau .co.id
!
Situs meminta login ulang tanpa alasan jelas meski sudah login sebelumnya
!
Tampilan situs sedikit berbeda dari biasanya — font, layout, atau warna yang tidak pas
Cara browser menampilkan domain homograf (Punycode)

Browser modern (Chrome, Firefox, Edge) kadang mengonversi domain IDN yang mencurigakan ke format Punycode untuk memperlihatkan karakter sebenarnya:

Terlihathttps://www.аpple.com
Sebenarnyahttps://www.xn--pple-43d.com
Uji kemampuanmu — pilih URL yang ASLI

Skor kamu:

Yang harus dilakukan
Selalu ketik URL secara manual untuk situs penting — bank, email, sistem internal kantor
Gunakan password manager — otomatis mendeteksi domain yang tidak cocok dan tidak akan auto-fill
Aktifkan MFA/2FA — walau password dicuri, akun tetap terlindungi dengan autentikasi dua faktor
Bookmark situs penting — gunakan bookmark yang sudah diverifikasi, bukan klik dari link email
Update browser secara rutin — browser terbaru memiliki proteksi IDN homograf yang lebih baik
Gunakan DNS filtering — solusi seperti Cisco Umbrella atau Cloudflare Gateway memblokir domain berbahaya
Laporkan ke IT Security jika menerima link mencurigakan via email atau chat internal
Yang tidak boleh dilakukan
✕
Jangan klik langsung link dari email, SMS, atau chat yang tidak diminta
✕
Jangan percaya gembok SSL saja sebagai bukti situs aman — domain palsu bisa punya SSL valid
✕
Jangan login di situs yang didapat dari link email tanpa memverifikasi URL secara hati-hati
✕
Jangan abaikan peringatan browser tentang situs berbahaya atau tidak aman
✕
Jangan share link internal kantor via saluran tidak resmi yang bisa disadap atau dimodifikasi
✕
Jangan mengira tampilan situs yang sama sudah cukup membuktikan URL-nya asli
✕
Jangan masukkan kredensial atau data sensitif sebelum memverifikasi domain dengan benar
Tools bantu deteksi homograf
Browser built-in — Chrome & Firefox menampilkan Punycode untuk domain mencurigakan
VirusTotal — scan URL untuk cek reputasi dan potensi bahaya
URLVoid — cek reputasi domain dari berbagai sumber blacklist
Unicode Inspector — tempel URL untuk melihat kode karakter sesungguhnya
DNS filtering (Cloudflare / Umbrella) — proteksi di level jaringan kantor
Email security gateway — scan dan blokir link berbahaya sebelum sampai ke inbox
Prinsip utama: Jika ragu, jangan klik — ketik manual atau tanya IT Security
Satu klik pada URL palsu bisa membahayakan seluruh organisasi
