Penyimpanan & Akses
Klasifikasikan setiap dokumen — Public, Internal, Confidential, atau Strictly Confidential
Terapkan prinsip least privilege — akses hanya sebatas kebutuhan pekerjaan
Simpan di lokasi resmi — server kantor, SharePoint, atau Drive korporat yang dikelola IT
Enkripsi file sensitif sebelum dikirim, dan gunakan password via saluran berbeda
Backup data secara rutin di lokasi aman dan terpisah dari penyimpanan utama
Pengiriman & Distribusi
Gunakan email korporat untuk semua komunikasi dokumen internal dan eksternal
Verifikasi penerima sebelum mengirim — pastikan alamat email sudah benar
Audit log akses dokumen kritikal — catat siapa, kapan, dan untuk apa
Gunakan nama file standar — format [Tahun]_[Dept]_[Nama]_[Versi] agar mudah dilacak
Set expiry link saat berbagi file via cloud — jangan biarkan link aktif selamanya
Fisik & Clean Desk
Terapkan clean desk policy — kunci semua dokumen saat meninggalkan meja
Musnahkan dokumen tidak terpakai dengan shredder — jangan buang utuh di tempat sampah
Ambil langsung hasil cetak/fotokopi — jangan biarkan dokumen tertinggal di mesin
Lock layar komputer setiap kali meninggalkan meja, bahkan sebentar sekalipun
WFH & Tempat Umum
Gunakan VPN saat mengakses sistem kantor dari jaringan luar
Pasang privacy screen filter saat bekerja di tempat umum atau co-working space
Gunakan koneksi aman — hindari WiFi publik tanpa VPN untuk akses dokumen rahasia
Pastikan anggota keluarga tidak bisa melihat atau mengakses dokumen kerja dari rumah
Penyimpanan & Akses
✕
Jangan simpan dokumen perusahaan di Google Drive / storage pribadi yang tidak dikelola IT
✕
Jangan copy source code atau konfigurasi server ke perangkat pribadi
✕
Jangan share password, API key, token, atau credential melalui chat atau email
✕
Jangan berikan akses dokumen kepada pihak yang tidak berkepentingan, termasuk rekan kerja
✕
Jangan simpan network diagram atau IP address di tempat yang bisa diakses publik
Pengiriman & Komunikasi
✕
Jangan kirim dokumen sensitif via WhatsApp, Telegram, atau chat pribadi
✕
Jangan forward email berisi data rahasia ke email pribadi dengan alasan apapun
✕
Jangan balas email ke penerima yang salah — selalu cek ulang sebelum klik Send
✕
Jangan klik link atau lampiran dari email mencurigakan yang meminta akses dokumen
✕
Jangan share link cloud tanpa batas waktu — selalu set expiry dan batasi akses
Data IT & Source Code
✕
Jangan upload source code ke repository publik (GitHub public, Pastebin, dll)
✕
Jangan hardcode API key, token, atau password langsung di dalam kode
✕
Jangan share network diagram atau topologi jaringan ke luar tanpa persetujuan
✕
Jangan paste source code internal ke AI tools publik seperti ChatGPT tanpa approval IT
Fisik & WFH
✕
Jangan buang dokumen fisik kritikal tanpa dihancurkan terlebih dahulu dengan shredder
✕
Jangan tinggalkan dokumen rahasia terbuka di meja saat pergi dari tempat duduk
✕
Jangan foto atau screenshot dokumen rahasia untuk dikirim ke luar perusahaan
✕
Jangan cetak dokumen confidential di printer umum atau printer rumah
Panduan keamanan per media penyimpanan
📄 Fisik (Kertas & Binder)
Simpan di lemari terkunci — pisahkan antara dokumen biasa dan confidential
Beri label klasifikasi yang terlihat jelas di setiap dokumen fisik
Catat peminjaman dokumen — siapa yang membawa dan kapan dikembalikan
✕
Jangan biarkan dokumen tergeletak di meja terbuka atau area umum
✕
Jangan buang tanpa shredder — robek manual tidak cukup aman
💻 Digital (Laptop & PC)
Aktifkan enkripsi disk penuh (BitLocker / FileVault) di semua perangkat kerja
Lock layar otomatis setelah 5 menit tidak aktif
Laporkan ke IT jika perangkat hilang atau dicuri — segera dan jangan tunda
✕
Jangan gunakan perangkat pribadi untuk dokumen perusahaan tanpa MDM
✕
Jangan colokkan flash disk tidak dikenal ke perangkat kantor
☁ Cloud (Drive & SharePoint)
Gunakan hanya platform cloud yang sudah disetujui IT perusahaan
Set permission sharing seminimal mungkin — view only jika memungkinkan
Review secara berkala siapa saja yang masih punya akses ke folder/file penting
✕
Jangan upload dokumen confidential ke akun cloud pribadi
✕
Jangan set folder kantor menjadi "Anyone with the link can view"
✉ Email
Selalu cek ulang alamat penerima sebelum mengirim dokumen sensitif
Enkripsi attachment berisi data kritikal, kirim password via saluran terpisah
Tandai email berisi info confidential dengan label atau subject yang jelas
✕
Jangan reply-all jika tidak semua penerima perlu tahu isi dokumen
✕
Jangan forward email internal berisi data sensitif ke email pribadi
Langkah pelaporan insiden kebocoran dokumen
1
Identifikasi & jangan panik
Catat apa yang terjadi — dokumen apa, kapan, dan bagaimana bisa bocor atau hilang. Jangan coba sembunyikan atau tangani sendiri.
2
Laporkan segera ke atasan & tim IT/Security
Waktu adalah kunci. Semakin cepat dilaporkan, semakin cepat bisa ditangani. Gunakan jalur pelaporan resmi yang disediakan perusahaan.
3
Jangan hapus atau ubah bukti
Pertahankan semua evidence — log akses, email, file, screenshot. Ini penting untuk investigasi dan menentukan root cause.
4
Batasi penyebaran
IT akan membantu mencabut akses, menonaktifkan link yang bocor, atau melakukan remote wipe jika perangkat hilang.
5
Dokumentasikan & tindak lanjut
Ikuti proses investigasi bersama tim Security. Setelah selesai, pelajari lesson learned agar insiden serupa tidak terulang.
Skenario yang wajib segera dilaporkan
!
Email berisi dokumen rahasia terkirim ke alamat yang salah
!
Laptop atau flash disk berisi data perusahaan hilang atau dicuri
!
Menemukan dokumen confidential tersebar di tempat umum atau media sosial
!
Ada akses tidak sah ke folder atau sistem yang bukan wewenang seseorang
!
Source code atau konfigurasi jaringan ditemukan di repository publik
!
Menerima permintaan dokumen dari pihak mencurigakan mengatasnamakan perusahaan
Prinsip utama: Informasi yang tepat → Orang yang tepat → Tempat yang tepat → Waktu yang tepat
Jika ragu, tanya tim IT/Security sebelum berbagi atau menyimpan dokumen apapun
